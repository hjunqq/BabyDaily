# BabyDaily 设计更新与 Gemini 验证清单

## 设计方向
- 调性：可爱/温柔，降低拟物度，采用半扁平+软阴影，保留 1–2 个玻璃亮点（头像/主按钮）。
- 色彩：主粉 #FFB7C5；文字主色 #5A3A2E；中性浅背景 #FFF7F9 或 #F9F5F3；对比色不超过 1–2 个（如浅青 #E0FFFF 仅做点缀）。
- 字体：标题用圆体 Display 22–24px 700；正文用 Sans 14–16px 500–600，行高 1.4–1.6。

## 移动端 Home vNext（Sakura Baby Tracker）
1) 结构
- 顶部：头像卡可做轻玻璃；标题与副标题（宝宝名/日期）。
- 关键总览：今日奶量/尿布/睡眠三个小卡片（半扁平，深色文字，次级说明行）。
- 列表：仅显示最近 3–5 条，左对齐时间与标签，数值加粗；底部“查看全部”。
- 主操作：悬浮/固定的“+ 记录”主按钮（粉色主色，明显按压态）；次要操作放动作面板或小圆角按钮。

2) 视觉与可读性
- 背景：单一淡粉/浅米渐变或轻模糊，减少花瓣数量，避免干扰文字。
- 按钮/卡片：半扁平或轻浮雕，内嵌纯色矢量图标，保证小尺寸清晰；统一圆角与 8–12px 内边距。
- 对比度：文本与图标需满足 WCAG AA；正文用深棕/深灰，按钮文字加粗。

3) 交互与状态
- 按压/禁用/加载状态清晰；提供加载骨架与空态插画提示“去记录”。
- 触控尺寸 ≥44px；图标有 aria-label/辅助说明。

## Web Dashboard vNext（Little Blossom Tracker）
1) 版面结构
- 顶部：搜索 + 轻量状态区（通知/语言/用户），弱化背景光晕。
- KPI 卡：3–4 张粉色系卡，文字加深、标题加粗，副文案次级色；底部基线对齐。
- 趋势图：带坐标轴、网格线、单位；双配色区分奶量/辅食；减少纯装饰点。
- 表格：表头有分隔线/底纹，行距充足，操作列用明确按钮或省略号菜单+提示。
- 导航：侧栏图标扁平化，选中态用左侧高亮条+色块，减少碎花与重阴影。

2) 视觉与可用性
- 主内容背景用浅米/浅灰，减少通栏粉；留白充足；图标/文字对比度提升。
- 组件圆角与阴影统一；悬停/聚焦/禁用状态可见；可点击区域 ≥44px。

3) 信息与文案
- KPI 副文案一致格式（例：Night: 6h · Naps: 2h 30m）。
- 语言切换与通知保持轻量但可辨；图标有 aria-label/tooltip。

## 交付要求给 Gemini
- 输出两个版本：A（在现有风格上微调）、B（半扁平清爽版）。
- 保留可爱调性，但优先可读性与可用性；文本对比度需达 WCAG AA。
- 提供：高保真稿（移动 Home + Web Dashboard）、切图/图标规范、色板与字阶、组件状态（默认/悬停/按压/禁用/加载/空态/错误）。

## 验证清单（验收 Gemini 输出）
- 对比度：文本/图标对背景均满足 WCAG AA；主按钮文字足够深。
- 层级：关键总览（移动）和 KPI 卡（Web）在视觉层级上清晰；主操作突出且唯一。
- 视觉简化：花瓣/光晕/玻璃数量明显减少，仅保留 1–2 个亮点；图标为纯色矢量，非重玻璃。
- 布局：移动列表最近 3–5 条且有“查看全部”；Web KPI 基线对齐，表格有表头分隔；趋势图有坐标/网格/单位并区分奶量与辅食。
- 组件与状态：圆角/阴影/内边距一致；按钮/卡片有悬停/按压/禁用；提供加载骨架与空态。
- 可用性：触控与点击区域 ≥44px；导航选中态清晰；操作列按钮或菜单可辨并有提示。
- 交付物：提供 A/B 两版；色板、字阶、图标规范与导出；关键屏的可下载资源。
