# BabyDaily 需求分析文档

## 1. 项目概述
**项目名称**: BabyDaily (宝宝日常)
**目标**: 开发一个全面的应用程序，用于记录宝宝的日常活动、健康状况和成长里程碑。该系统旨在促进家庭成员共同参与育儿，并为孩子的成长留下数字化的美好回忆。
**平台**:
- **微信小程序**: 主要界面，方便家庭成员进行日常记录和快速查看。
- **Web 前端**: 管理后台，提供数据可视化和详细的系统配置功能。
- **后端 API**: 中央服务器，处理数据存储、用户认证和业务逻辑。

## 2. 用户角色
1.  **管理员 (父母)**: 拥有所有功能的完全访问权限，包括邀请家庭成员、管理数据和系统配置。
2.  **家庭成员 (祖父母、亲戚)**: 可以查看和添加日常记录。无法访问敏感的系统设置。
3.  **访客 (可选)**: 仅拥有特定共享内容（如照片、里程碑）的只读访问权限。

## 3. 功能需求

### 3.1. 日常记录 (婴幼儿阶段)
*目标用户: 所有角色*
- **喂养**:
    - 母乳喂养 (时间, 时长, 左右侧)。
    - 瓶喂 (时间, 奶量, 类型: 配方奶/母乳)。
    - 辅食 (时间, 食物名称, 摄入量)。
- **换尿布**:
    - 时间, 类型 (嘘嘘/便便/两者都有), 性状 (颜色, 质地 - 可选)。
- **睡眠**:
    - 开始时间, 结束时间, 时长。
    - 睡眠质量备注。
- **健康与卫生**:
    - 洗澡记录。
    - 体温记录 (生病时)。
    - 用药记录。

### 3.2. 健康与成长管理
*目标用户: 管理员, 家庭成员*
- **生长曲线**:
    - 记录身高, 体重, 头围。
    - 对比 WHO/标准生长曲线进行可视化展示。
- **疫苗接种计划**:
    - 标准疫苗清单。
    - 即将接种疫苗的提醒功能。
    - 记录实际接种日期和接种反应。
- **医疗记录**:
    - 就医记录, 处方, 症状描述。

### 3.3. 成长与里程碑 (幼儿及儿童阶段)
*目标用户: 管理员, 家庭成员*
- **里程碑追踪**: 第一次微笑, 第一次走路, 第一次说话等 (支持上传照片/视频)。
- **日常任务/习惯**:
    - 大龄儿童的打卡清单 (例如: 刷牙, 读书)。
    - 奖励系统 (星星/积分)。
- **日记/日志**: 自由文本记录，用于记录难忘时刻。

### 3.4. 系统与用户管理
*目标用户: 管理员*
- **家庭管理**:
    - 通过微信邀请用户。
    - 权限管理。
- **数据导出**: 将数据导出为 PDF 或 Excel 进行备份。
- **档案**: 宝宝档案 (姓名, 出生日期, 血型, 过敏史)。

## 4. 非功能性需求
- **易用性**:
    - 小程序必须对老年家庭成员极其友好（大按钮，清晰的文字）。
    - 针对高频操作提供快速记录模式（例如：“刚拉了”一键按钮）。
- **性能**:
    - 小程序加载速度要快。
    - 离线记录能力（联网后同步）是加分项，但 V1 版本可选。
- **安全性**:
    - 安全认证 (微信登录)。
    - 数据隐私至关重要；照片和个人数据必须得到保护。
- **可扩展性**:
    - 后端应支持高效存储媒体文件（照片/视频）（例如：对象存储）。

## 5. 建议架构
- **前端 (Web)**: React 或 Vue.js (单页应用 SPA)。
- **小程序**: 微信原生开发 或 Uni-app (具有跨平台潜力)。
- **后端**: Node.js (NestJS/Express) 或 Python (Django/FastAPI) 或 Go。
- **数据库**: PostgreSQL 或 MySQL (用于记录的关系型数据), Redis (缓存)。
- **存储**: 云对象存储 (OSS/S3) 用于媒体文件。

## 6. 未来规划
- AI 分析: 睡眠模式分析, 生长预测。
- 社区: 与其他父母分享育儿经验 (可选, 低优先级)。
