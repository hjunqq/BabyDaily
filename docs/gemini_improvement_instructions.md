# BabyDaily 原型改进指令（提供给 Gemini）

## 移动端 Home（Sakura Baby Tracker）
- 视觉简化：减少玻璃和发光，背景用单一淡粉/浅米渐变或轻微模糊，仅保留头像或主操作的轻微玻璃高光。
- 层级突出：在列表上方增加今日关键总览（今日奶量、今日尿布、今日睡眠），列表只显示最近 3–5 条，加“查看全部”。
- 主要操作：四个大圆钮合并为“+ 记录”主按钮（粉色主色，明显按压态），其他操作放进弹出动作面板或次要小圆角按钮。
- 对比度：正文文字改为更深的棕/深灰（对比度 ≥ WCAG AA），按钮/卡片文字加粗；弱化背景花瓣数量，避免干扰可读性。
- 图标样式：圆形按钮改为半扁平/轻浮雕，内嵌图标用纯色矢量（非玻璃），确保小尺寸清晰。
- 版式节奏：时间、标签左对齐，数值加粗；卡片内部增加 8–12px 的内边距，统一圆角；分隔线或微弱阴影提升可扫描性。
- 反馈与状态：为按钮/卡片提供按压/悬停（桌面）/禁用状态；增加加载骨架和空态插画提示“去记录”。

## Web Dashboard（Little Blossom Tracker）
- 层级与对比：减少通栏粉色背景，主内容区用浅米/浅灰，关键 KPI 卡保持粉色系但文字加深、标题加粗，副文案用次级色。
- 导航优化：侧栏图标扁平化、线性化，选中态用左侧高亮条+色块；减小装饰元素数量，保持清晰。
- 文本与表格：提高正文对比度和字重，表格表头做分隔线/底纹，操作列改成明确的按钮/省略号菜单带提示。
- 图表信息性：给趋势图加坐标轴、网格线和单位；用两种有区分的色彩表示奶量/辅食，减少纯装饰点缀。
- 间距与对齐：统一卡片内的竖向间距和基线，使三个 KPI 卡的标题/数字/副文案对齐；减少背景碎花/光晕，留白更呼吸。
- 状态反馈：悬停、聚焦、禁用态可见；空态/错误/加载态有清晰文案和轻插画。
- 可用性：保证按钮和行高可点区域 ≥44px；图标都带 aria-label/tooltip；多语言切换和通知图标保持轻量但可辨。

## 设计系统与色/字建议
- 调色板：保留主粉（如 #FFB7C5），增加更深文字色（如 #5A3A2E）和中性浅背景（如 #FFF7F9 或 #F9F5F3），限定强调色 1–2 个。
- 字体层级：标题用圆体 Display（22–24px, 700），正文用清晰 Sans（14–16px, 500–600），行高 1.4–1.6。
- 风格方向：从重度拟物玻璃转为半扁平 + 软阴影，保留 1–2 个玻璃亮点（如头像卡或主按钮），其余保持干净。

## 交付要求
- 请输出两个版本：A（在现有风格上微调）、B（半扁平清爽版）。
- 保留可爱调性，但以可读性、可用性优先；确保文本对比度符合 WCAG AA。
